## 项目概述

**项目名称**: 云集智能文件清理专家
**核心功能**:
- 多类型垃圾文件智能扫描（迅雷临时文件、系统缓存、浏览器缓存等）
- 文件分类展示与排序（按名称、路径、大小、修改时间、类型）
- 条件过滤与批量选择删除
- 自定义清理规则扩展
- 扫描进度实时显示与取消控制
- 版本在线更新检查

```
云集智能文件清理专家/

│       ├── launcher.py         # 入口文件（单实例控制 + 调用main.py）
│       ├── icon.ico            # 应用图标

│       ├── requirements.txt    # Python依赖
│       ├── version_history.json # 版本历史
│       └── cleanup_settings.json # 运行时配置（Git不管理）
│
│ # ========== Git不管理 - 构建相关 ==========
├── build/                      # ⭐ Git不管理 - 构建目录（按版本组织）
│   ├── build.py                # 构建脚本模板
│   └── v2026.05.10/            # 版本构建记录
│       ├── build.py            # 该版本的构建脚本备份
│       ├── build/              # PyInstaller临时文件
│       └── dist/               # PyInstaller输出
│
│ # ========== Git不管理 - 其他 ==========
├── BAK/                        # ⭐ Git不管理 - 备份文件（旧版代码）
└── ver/                        # ⭐ Git不管理 - 稳定版发布
    └── version.json            # 版本信息（Git管理）
```

### ⚠️ 重要：Git管理范围与目录职责

| 目录 | Git管理 | 用途 | 说明 |
|------|---------|------|------|
| **dev/app/** (源码) | ✅ **Git管理** | 项目源代码 | Python源码、图标、依赖声明 |
| **docs/** | ✅ **Git管理** | 项目文档 | 开发指南等 |
| **ver/version.json** | ✅ **Git管理** | 版本信息 | 在线更新检查用 |
| **dev/*.exe** | ❌ **Git不管理** | EXE文件 | 体积大，通过构建脚本生成 |
| **build/** | ❌ **Git不管理** | 构建目录 | 按版本组织，方便回滚 |
| **BAK/** | ❌ **Git不管理** | 备份文件 | 旧版本备份 |
| **app/cleanup_settings.json** | ❌ **Git不管理** | 运行时配置 | 用户本地设置 |

### dev/ 目录整洁原则

`dev/` 目录下**只放 EXE 文件**，所有其他文件归类到 `app/` 子目录：

| 文件类型 | 存放位置 | 说明 |
|----------|----------|------|
| EXE文件 | `dev/*.exe` | 构建产物 |
| 源代码/资源 | `dev/app/` | Python源码、图标、依赖声明 |
| 运行时配置 | `dev/app/cleanup_settings.json` | 用户本地设置 |
| Python缓存 | `dev/app/__pycache__/` | 自动生成，.gitignore已忽略 |

### build/ 目录版本管理

构建目录按版本号组织，每个版本独立保存构建脚本和临时文件：

```
build/
├── build.py                    # 当前使用的构建脚本模板
├── v2026.05.10.1600/           # 版本构建记录
│   ├── build.py                # 该版本构建脚本备份（可用于回滚）
│   ├── build/                  # PyInstaller工作目录
│   └── dist/                   # PyInstaller输出目录
└── v2026.05.11.1000/           # 另一个版本
    ├── build.py
    ├── build/
    └── dist/
```

**回滚方法**：如需回滚到某版本，使用该版本目录下的 `build.py` 重新构建即可。

## 核心文件说明

### 🚀 应用目录 (`dev/app/`)

| 文件/目录 | 用途 | 说明 |
|-----------|------|------|
| `dev/app/main.py` | 主程序 | CustomTkinter GUI，文件扫描/清理核心 |
| `dev/app/launcher.py` | 入口文件 | 单实例控制 + 调用 main.py 的 main() |
| `dev/app/icon.ico` | 应用图标 | Windows图标 |
| `dev/app/icon.png` | 高清图标 | PNG格式图标 |
| `dev/app/requirements.txt` | Python依赖 | customtkinter等 |
| `dev/app/version_history.json` | 版本历史 | 构建记录 |
| `dev/app/cleanup_settings.json` | 运行时配置 | 用户本地设置（Git不管理） |

### 🔧 构建脚本 (`build/build.py`)

**构建命令**：
```bash
cd build/
python build.py
```

**构建流程**：
1. 动态生成版本号：`YYYY.MM.DD.HHMM`
2. 在 `build/v版本号/` 下创建版本构建目录
3. 将版本号注入 `dev/app/main.py` 的 VERSION 变量
4. PyInstaller 从 `dev/app/launcher.py` 构建 EXE
5. EXE 输出到 `dev/` 目录，文件名 `云集智能文件清理专家-vYYYY.MM.DD.HHMM.exe`
6. 构建完成后恢复源代码中的版本号
7. 自动清理旧版本 EXE
8. 将构建脚本备份到 `build/v版本号/build.py`，方便回滚

## 开发工作流

### ⭐ 标准开发流程

```
1. 日常开发 → 在 dev/app/ 修改和测试
   - 直接修改 main.py 等文件
   - 运行 python dev/app/launcher.py 测试

2. 构建EXE → 运行构建脚本
   cd build/
   python build.py

3. 测试EXE → 运行生成的EXE验证

4. Git提交 → 提交 dev/app/ 目录
   git add dev/app/
   git commit -m "v版本号: 修改描述"
   git push
```

### 🚀 稳定版发布流程（必须执行）

**当新版本放入 `ver/` 目录作为稳定版时，必须同步推送到远程仓库，确保用户能通过版本更新功能获取最新版本。**

```
1. 构建EXE → cd build/ && python build.py

2. 测试验证 → 运行EXE确认功能正常

3. 放入ver/ → 将EXE复制到 ver/ 目录

4. 更新version.json → 更新 ver/version.json 中的 latest 和 versions

5. Git提交并推送（必须！）
   git add dev/app/ ver/
   git commit -m "v版本号: 稳定版发布"
   git push

⚠️ 严禁：将EXE放入ver/后不推送到远程仓库！
   这会导致用户通过版本更新功能无法获取最新版本。
```

**稳定版发布检查清单：**

| 步骤 | 命令/操作 | 必须 |
|------|-----------|------|
| 构建 | `cd build/ && python build.py` | ✅ |
| 测试 | 运行EXE验证功能 | ✅ |
| 复制到ver/ | 将EXE放入ver/目录 | ✅ |
| 更新version.json | 更新latest和versions | ✅ |
| Git提交 | `git add dev/app/ ver/` | ✅ |
| Git推送 | `git push` | ✅ |

### 📝 Git 提交信息规范

**提交信息格式：`版本号: 简要描述`**

**示例：**
```
v2026.05.10.1600: 新增自定义清理规则功能
v2026.05.10.1600: 优化大目录扫描性能
v2026.05.10.1600: 修复文件删除后列表未刷新问题
```

## 垃圾文件识别规则体系

项目内置多种垃圾文件识别规则，按类别组织：

| 规则类别 | 识别内容 | 示例扩展名 |
|----------|----------|------------|
| 迅雷临时文件 | 迅雷下载产生的临时/配置文件 | .ttd, .td, .tdcfg, .xltd |
| 通用临时文件 | 系统和编辑器临时文件 | .tmp, .temp, .bak, .swp |
| 缓存文件 | 各类应用缓存 | .cache, .db, .sqlite |
| 下载软件文件 | 未完成的下载文件 | .download, .part, .crdownload |
| 种子/磁链文件 | BT种子和链接文件 | .torrent, .magnet |
| 广告/追踪文件 | 广告和追踪数据 | .ad, .ads, .tracking |
| 电驴下载文件 | eMule相关文件 | .part.met, .corrupt |
| 系统临时文件 | Windows系统临时文件 | Thumbs.db, Desktop.ini |
